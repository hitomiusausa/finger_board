# BoardScreen 機能仕様書

**作成日:** 2026-02-28  
**ステータス:** 確定（随時更新）  
**対象 Phase:** Phase 1〜2  
**関連ファイル:** `features/board/screens/board_screen.dart`

---

## 1. 概要

BoardScreen は Finger Board の中核となる画面。教材の1ページ（カード）を表示し、編集・提示・学習の3モードを切り替えて使用する。旧アプリの「カード編集画面」に相当する。

### 用語対応

| 旧アプリ | Flutter | 説明 |
|---|---|---|
| ファイル | material（教材） | 複数ページを含む単位 |
| カード | page（ページ） | ボードの1画面 |
| オブジェクト | BoardObject | ページ上に配置する要素 |
| カード一覧パネル | PageListPanel | ページ一覧（サイドパネル） |
| オブジェクト追加パネル | ObjectAddPanel | オブジェクト追加UI |
| ツールパネル | ToolPanel | オブジェクト編集UI |

## 2. モード定義

```dart
enum BoardMode {
  edit,     // 編集モード
  present,  // 提示モード
  study,    // 学習モード
}
```

### 2-1. 編集モード（edit）

教材を作成・編集するためのモード。全ての編集機能が有効。

**タッチ操作の振る舞い：**
- オブジェクトをタップ → ツールパネルを表示
- オブジェクトをドラッグ → 移動（位置変更）
- 空きスペースをダブルタップ → テキスト追加パネルを表示
- 空きスペースをスライド → 範囲選択

**有効な機能：**
- オブジェクト追加（テキスト・画像・線・AssembleBox）
- オブジェクト移動・削除・サイズ変更・回転
- オブジェクトのコピー・ペースト・複製
- グループ化・グループ解除
- 重ね順変更・整列
- ページ追加・削除・並べ替え・コピー
- 教材の保存（Supabase）
- アンドゥ・リドゥ

**AppBar の構成：**
```
[ファイルライブラリへ戻る] [教材タイトル] [提示モードボタン] [学習モードボタン] [保存ボタン]
```

### 2-2. 提示モード（present）

授業中に先生がプロジェクターや画面で操作しながら生徒に見せるモード。提示用に最適化。

**タッチ操作の振る舞い：**
- オブジェクトをタップ → タッチ機能を直接発動（音声再生・リンク先へ移動など）
- オブジェクトを長押し → 簡易編集メニューを表示（限定的な編集が可能）
- オブジェクトのドラッグ → 自由移動可能（編集モードと同じ）

**有効な機能：**
- タッチ機能の発動（音声・動画・リンク）
- QuestionBox の答えを表示する（正解を見せる側）
- ページ切り替え
- 限定的な編集（長押しメニュー経由）

**無効な機能：**
- オブジェクト追加
- 保存ボタン

**AppBar の構成：**
```
[編集モードに戻る] [教材タイトル] [学習モードボタン]
```

### 2-3. 学習モード（study）

生徒が自己学習・課題をこなすためのモード。旧アプリの「for Students」とほぼ同じ振る舞い。

**タッチ操作の振る舞い：**
- オブジェクトをタップ → タッチ機能を直接発動
- マグネットボックス（QuestionBox）内のオブジェクトをドラッグ → ボックス内でのみ移動可能
- カード直下のオブジェクトはドラッグ移動不可（原則）
- 複製ドラッグオブジェクトは指定されたボックス内にのみドロップ可能

**有効な機能：**
- タッチ機能の発動
- QuestionBox への解答（ドラッグ・テキスト入力）
- ページ切り替え
- 進捗の記録（Phase 2）

**無効な機能：**
- オブジェクトの追加・削除
- オブジェクトの編集
- 保存ボタン
- モード切替ボタン（for Students アプリでは非表示）

**AppBar の構成（Pro アプリ）：**
```
[編集モードに戻る] [教材タイトル] [提示モードボタン]
```

**AppBar の構成（for Students アプリ）：**
```
[ホームに戻る] [教材タイトル]
```

## 3. モード状態遷移

```
Pro アプリ起動
  └─ BoardScreen（編集モード）
       ├─ 提示モードボタン → 提示モード
       │     └─ 戻るボタン → 編集モード
       │     └─ 学習モードボタン → 学習モード
       └─ 学習モードボタン → 学習モード
             └─ 戻るボタン → 編集モード

for Students アプリ起動
  └─ BoardScreen（学習モード固定）
       └─ モード切替UI なし
```

## 4. オブジェクト種別と各モードでの振る舞い

| オブジェクト | 編集 | 提示 | 学習 |
|---|---|---|---|
| LetterBox（テキスト） | 編集・移動可 | タッチ機能発動 | タッチ機能発動 |
| ImgBox（画像） | 編集・移動可 | タッチ機能発動 | タッチ機能発動 |
| AssembleBox（SVG） | 編集・移動可 | タッチ機能発動 | タッチ機能発動 |
| Line（線） | 編集・移動可 | 表示のみ | 表示のみ |
| QuestionBox（問題）※Phase 2 | 設定 | 答えを表示 | 答える |
| GroupObject（グループ）※Phase 2 | グループ編集 | アニメーション | 学習用移動 |

## 5. ページ操作

### 5-1. ページ一覧パネル

- 画面左端（または右端）のハンドルをドラッグまたはタップで表示
- ページのサムネイルが一覧表示される
- サムネイルをタップで該当ページにジャンプ

### 5-2. ページ追加・削除

**編集モードのみ有効：**
- 「＋」ボタンで現在のページの次に白紙ページを追加
- ページをスワイプ → 削除ボタン表示
- 長押しでページ名変更

### 5-3. ページ移動

- 前後ボタン（▲▼）で1ページずつ移動
- ページ一覧パネルのサムネイルをタップでジャンプ

## 6. アンドゥ・リドゥ

**編集モードのみ有効。**

対象操作：
- オブジェクトの追加・削除
- オブジェクトの移動
- オブジェクトの外観変更（サイズ・色・フォント）
- グループ化・グループ解除

実装方針：`UndoManager` クラスで Command パターンにより管理。

## 7. 学習モードでのオブジェクト移動制限（重要）

旧アプリから踏襲する重要な仕様。Phase 2 の QuestionBox 実装時に必ず反映する。

```
移動できる           移動できない
─────────────────────────────────────────
アクティブな          カード直下に
QuestionBox内の      直接配置された
オブジェクト          オブジェクト
    ↓                    ↓
QuestionBox内で        ✕ 移動不可
自由に移動可
```

**複製ドラッグの特別ルール：**
- 複製ドラッグされたオブジェクトはカード直下にはドロップ不可
- QuestionBox 内のみドロップ可能

**例外：** `学習モード制限解除` 設定がONのオブジェクトはカード直下の移動も可能。

## 8. タッチ機能（Phase 2 以降で本格実装）

提示モード・学習モードでオブジェクトをタッチした時に発動する機能。

| タッチ機能 | 内容 | Phase |
|---|---|---|
| 音声再生 | 設定した音声ファイルを再生 | 2 |
| 動画再生 | 設定した動画ファイルを再生 | 2 |
| ページリンク | 指定したページに移動 | 2 |
| 書類表示 | PDF等の書類を表示 | 2 |
| 音声読み上げ | オブジェクトの名前を TTS で読み上げ | 3 |

---

## 9. 実装上の注意点

- `BoardMode` は `BoardState` に持たせ、`boardProvider` 経由で参照する
- for Students アプリでは `BoardScreen` を `BoardMode.study` 固定で起動する
- モード切替時にオブジェクトの選択状態はリセットする
- 学習モードでの進捗記録は Phase 2 で追加する（Phase 1 では記録しない）
